What is CI/CD?

CI/CD stands for Continuous Integration and Continuous Deployment. Continuous Integration (CI) is the practice of automating the integration of code changes from multiple contributors into a single repository. Continuous Deployment (CD) goes a step further by automatically deploying those validated changes to a live environment. Together, they form a pipeline that helps development teams deliver software updates faster, more reliably, and with fewer manual errors.

My Pipeline

When I push my code changes to the main branch on GitHub, a GitHub Actions workflow is automatically triggered. This workflow checks out my code, verifies that all required files exist, and then automatically deploys the static files to GitHub Pages, making the updated website live on the internet.
